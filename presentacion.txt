════════════════════════════════════════
LÁMINA 4 — ¿QUÉ ES UN LLM? (para no técnicos)
════════════════════════════════════════
Título: ¿Qué hay detrás del asistente?

Imaginá un empleado que leyó millones de documentos,
entiende cualquier pregunta en español, y puede consultar
sistemas internos para responderte.

Eso es un LLM — Large Language Model.

- No memoriza respuestas — las construye en el momento
- Entiende contexto, sinónimos y lenguaje coloquial
- GPT-4o es el modelo más avanzado de OpenAI (el mismo detrás de ChatGPT)
- Nosotros lo conectamos a los datos reales de Santander


════════════════════════════════════════
LÁMINA 5 — ARQUITECTURA
════════════════════════════════════════
Título: ¿Cómo llega la respuesta al celular?

Crea un diagrama de flujo vertical con estas 6 etapas:

Etapa 1: Ejecutivo Comercial
El ejecutivo escribe una pregunta por WhatsApp

Etapa 2: WhatsApp (Meta)
Recibe el mensaje y lo entrega al servidor

Etapa 3: Servidor (FastAPI)
Recibe el mensaje y coordina el proceso

Etapa 4: GPT-4o (OpenAI)
Entiende la pregunta y decide qué datos necesita.
Se conecta a la API de Santander para obtener datos reales.

Etapa 5: Respuesta generada
GPT-4o redacta la respuesta en español o genera una gráfica

Etapa 6: Ejecutivo Comercial
Recibe la respuesta en WhatsApp en segundos

Notas al costado del diagrama:
- El ejecutivo nunca sale de WhatsApp
- GPT-4o decide solo qué datos necesita consultar
- La API de Santander ya existía — no se modificó


════════════════════════════════════════
LÁMINA 6 — SEGURIDAD POR CAPAS
════════════════════════════════════════
Título: GPT-4o nunca toca la base de datos

Crea un diagrama de 3 capas horizontales apiladas (de arriba a abajo):

CAPA EXTERNA — IA (GPT-4o)
Solo habla con funciones controladas.
No tiene acceso directo a ningún sistema interno.

CAPA INTERMEDIA — API Santander
Expone solo los datos que decide exponer.
Actúa como filtro y guardián de la información.

CAPA INTERNA — Base de Datos Santander
Nunca es accedida directamente por la IA.
Protegida detrás de la API.

Nota importante en la lámina:
"GPT-4o no conoce la estructura de la base de datos,
no puede hacer consultas directas, no puede modificar nada.
Solo puede llamar a funciones específicas que nosotros definimos."

Por qué esto importa:
- La IA no puede filtrar datos que no le permitamos
- Si la IA es comprometida, la BD sigue protegida
- Cumple con estándares de seguridad bancaria


════════════════════════════════════════
LÁMINA 7 — FUNCTION CALLING (cómo piensa el agente)
════════════════════════════════════════
Título: ¿Cómo sabe el agente qué datos consultar?

La magia está en una técnica llamada "Function Calling":

EJEMPLO — el ejecutivo escribe:
"¿Cómo van los jefes comerciales este mes?"

EL AGENTE PIENSA:
  1. Entiendo que preguntan por jefes comerciales
  2. Tengo una herramienta llamada get_kpi_por_jefe
  3. El período es el mes actual: 202604
  4. Llamo a la API con esos parámetros
  5. Recibo los datos reales
  6. Genero una respuesta clara en español

HERRAMIENTAS DISPONIBLES:
  → get_kpi_total        : KPIs globales del mes
  → get_kpi_por_canal    : Desglose por canal de venta
  → get_kpi_por_lider    : Ranking de líderes de equipo
  → get_kpi_por_jefe     : Ranking de jefes comerciales
  → get_chart_jefes      : Gráfica de líneas por jefe
  → get_chart_lideres    : Gráfica de líneas por líder
  → get_chart_yoy        : Comparativa año vs año vs año

El agente elige la herramienta correcta según lo que el
usuario pregunta — sin que el usuario sepa que existen.
